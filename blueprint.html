Backend First Execution Plan

Build the backend in clear, testable parts

The project starts with a Docker-powered backend foundation. Each phase should produce something runnable, debuggable, and reviewable before moving forward. Once the API contracts are stable, the mobile frontend can connect with confidence.

🐳 Docker Development Loop
docker compose up→ API container→ Postgres + PostGIS→ Redis→ Prisma migrate→ Seed data→ Health check→ API tests
Part 1 — Repository & Docker FoundationStart Here
  • Create backend repo: backend-api with TypeScript, Express, linting, and env validation.
  • Add docker-compose services for api, postgres-postgis, redis, and optional prisma studio.
  • Add health endpoints: GET /health and GET /health/db for fast debugging.
  • Lock local commands: docker compose up, migrate, seed, test, and logs.
Part 2 — Database Design & PrismaSchema
  • Define base models: User, Profile, Role, OtpCode, RefreshToken, AuditLog.
  • Add pillar models gradually: experiences, events, business, seva, bhajans, groups, pilgrimage, donations.
  • Create migrations and seed data for local testing in Docker.
  • Keep Zod DTOs shared so API and mobile app use the same contract.
Part 3 — Auth, Security & RBACCore API
  • Implement mobile OTP send and verify APIs with Redis rate limiting.
  • Issue access and refresh JWT tokens after successful OTP verification.
  • Add auth middleware, role guards, helmet, cors, request logging, and error handling.
  • Test devotee, elder, mandir-admin, and super-admin access paths.
Part 4 — Pillar APIs in Priority OrderFeatures
  • Build first vertical slice: experiences feed with create, list, detail, moderation status.
  • Add events with PostGIS location support and near-me queries.
  • Add business directory, seva requests, bhajans/media, groups, pilgrimage, and donations.
  • Each module ships with routes, controller, service, Prisma queries, validation, and tests.
Part 5 — Jobs, Storage & NotificationsAsync
  • Use BullMQ workers for push notifications, media processing, reminders, and digest jobs.
  • Add Azure Blob upload flow with signed URLs and CDN-ready media records.
  • Store Expo push tokens and create notification preferences per user.
  • Keep worker containers separate from API containers in Docker compose.
Part 6 — Testing, Docs & Frontend HandoffReady for UI
  • Add unit tests for services and integration tests for REST endpoints.
  • Generate OpenAPI docs or a Postman collection for all stable endpoints.
  • Freeze request/response DTOs before connecting Redux Saga and Axios in the app.
  • Only then start frontend screens module by module against the real Docker API.
